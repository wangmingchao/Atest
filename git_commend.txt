git init
把当前目录变为git可管理的仓库

git add filename
把文件添加到缓存区

git commit -m "注释"
把文件提交到仓库

git status
查看此时仓库的状态

git diff filename
查看文件新修改的内容

git log
查看历史记录

git log --pretty=oneline
用简单的方式显示历史记录

git reset --hard HEAD^
回退到上一版本

git reset --hard HEAD^^
回退到上上一个版本（一次类推）

git reset --hard 版本号
恢复到指定的版本

git reflog
显示所有的版本（包括回退处理增加的版本）

git checkout -- filename
将指定文件恢复为缓存区的内容

rm filename
在工作区删除指定文件（与其它操作一样，然后执行add和commit操作提交此删除操作）

git remote add origin networkaddr
把两个仓库关联起来（本地仓库是现在所处的工作空间，远程为网络地址所指定的仓库）

git push origin 分支名
把本地分支推送到远程指定分支（第一次执行命令需要加参数-u：git push -u origin master）

git clone networkaddr
将远程库克隆到本地

git branch 分支名
创建分支

git checkout 分支名
切换分支

git checkout -b 分支名
创建并切换分支

git merge 分支名
将指定的分支合并到当前所在的分支
（git中解决冲突即用此命令合代码后，git会生成一个新的文件，该文件里包含两个分支里的修改内容，我们在这个文件里修改成我们想要的，然后保存就可以了）

git branch -d 分支名
删除指定分支

多人协作：
一、指定本地某分支与远程的某分支链接
例如将本地dev分分支与远程的origin/dev分支链接
git branch --set-upstream dev origin/dev
二、在本地创建某分支并和远程的某分支链接
git checkout -b dev origin/dev
三、多人协作主要解决的问题是冲突问题，一般先通过git pull命令从远程抓取最新代码，然后在本地解决冲突，然后再push
